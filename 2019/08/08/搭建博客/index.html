前言

我一直想搭一个自己的网站，奈何自己搞个服务器也不是很必要（我们穷鬼是这样的啦），那么用Hexo搭建一个博客，将代码托管到Github就显得很划算了。

建立这个博客的初衷还是想把自己学过的和实际使用过的技术进行复盘，督促自己不要变成咸鱼，主要是把自己摔过的坑记录下来，除非十分必要，否则过程从略记录。

要搭建这个博客大概需要以下的几个方面准备：

  1. 有一台电脑。
  2. 安装Hexo。
  3. Github账号。
  4. MarkDown编辑博文（感觉MarkDown的语法比Latax简洁很多啊！！！）。

各种软件的安装

Node安装

首先得在官网上把这个Node下载下来，大概就是无脑下载和安装就可以了（找好路径之后，一直点Next，最后Finish）。

Git的安装

下载Git也是大概无脑安装就行。下载完成之后去cmd里输入git version，如果出现了它的版本信息那就是没问题了。

在这个部分中，我始终没搞清楚Git的三个桌面快捷方式到底分别是干啥的，不过好在弄这个博客只用得到Git Bush。

与Github关联

前期工作

  1. 首先建立一个Github账户。
  2. 创建一个新的代码库。
  3. 名字要是<Github的用户名>.github.io

    配置SSH密钥

    我觉得这一块我真的没这么搞懂，但是做大概是做了出来的。

首先在Git Bush里使用ssh-keygen -t rsa -C "your_email@example.com"命令，里面的那一堆email地址换成自己的（双引号里面的）。

如果不想该默认存储地址的话，就一路回车，直到出现:

将本地博客上传到网上

包括主题的选择在内的各项内容在参考文献中说的非常清楚，下次再遇到问题可以直接使用里面的过程。

值得注意的是，我在使用这些内容的时候遇到的一些问题：

  1. 需要在根目录_config.yml文件下修改使用的主题名称，如果仅仅是把主题下载到theme文件夹是不行的。
  2. source里的_posts文件夹中所有.md文件都是MarkDown语法下的博文文件，要想编辑博客，增加或删改博文的话，就要对这些文件进行编辑，我使用的是VS Code软件配合MarkDown All In One插件使用。
  3. 进入下载的theme里面会还有一个_config.yml，在那个里面编辑的就是各个页面的壁纸或者名称之类的，这些壁纸的主题都要先下载到图片的文件夹里，然后再把路径贴到文件中就行，但是地址要写对，要不然很可能显示不出来。
