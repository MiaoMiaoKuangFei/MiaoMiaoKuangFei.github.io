Problem

Given a linked list, remove the n-th node from the end of list and return its head.

Example:

Given linked list:
$1→2→3→4→5$, and $n = 2$.

After removing the second node from the end, the linked list becomes $1→2→3→5$.

Note:
Given n will always be valid.

Follow up:
Could you do this in one pass?

My Solution

class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        ListNode shift=head;//Get the first Node
        /**get size*/
        int size=0;
        while(shift!=null){
            size+=1;
            shift=shift.next;
        }
        if(n==size) return head.next;//Check if we should delete the head Node
        else{
            shift=head;
            int step=size-n-1;
            while(step!=0){
                shift=shift.next;
                step-=1;
            }           
            shift.next=shift.next.next;
            return head;
        }
    }
}

Summary

The summary on leetcode shows that we have two ways to solve the problem. Whatever the method we use, the time complexity is always $O(L)$ and the space comlexity is always $O(1)$.

It comes out a problem that we needn’t to use so-called one pass method to save time or space .

And LeetCode provides two method as below:

  1. Approach 1: Two pass algorithm

    This is the approach i use.I create a pointer named shift,make it through out all the nodes and get the length of the list, therefore we can know what exactly step we should move and just move and say shift.next=shift.next.next;.However ,it contains a condition that $n=size$,which means we should return head.next instead .The approach from LeetCode is to build a dummy node at the front of head ,while mine is to discuss and say:

    if(n==size) return head.next

  2. Approach 2: One pass algorithm

    This approach is to create two points and the distance between them is $n$, so when the first pointer points to the end, the second pointer just point to the goal node.

    However, some doubt that this can not contribute to the efficiency at all, and the so-called one pass is not different to two pass algorithm which both use two pointers.
